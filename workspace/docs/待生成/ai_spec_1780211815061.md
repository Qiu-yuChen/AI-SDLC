# 图书管理系统产品规格说明书

## 1. 项目概述
### 1.1 背景
当前图书馆或小型图书收藏机构缺乏数字化管理工具，导致图书借阅、库存跟踪和读者管理效率低下。本项目旨在开发一套图书管理系统，实现图书信息的集中管理、借阅流程自动化以及数据统计分析，提升运营效率和用户体验。

### 1.2 目标
- 实现图书信息的增删改查，支持批量导入（如 CSV/Excel）。
- 支持读者（用户）注册、登录及个人信息管理。
- 提供图书借阅、归还、预约功能，并自动更新库存状态。
- 生成借阅记录、逾期提醒及统计报表（如热门图书、借阅趋势）。
- 提供管理员后台，用于权限管理和系统配置。

### 1.3 范围
- **包含**：图书管理、读者管理、借阅管理、统计报表、管理员后台。
- **不包含**：第三方支付集成、电子书在线阅读、跨图书馆联盟系统。

## 2. 功能需求
### 2.1 用户角色
| 角色 | 描述 |
|------|------|
| 管理员 | 管理图书、读者、借阅记录，配置系统参数，查看统计报表 |
| 普通读者 | 查询图书、借阅/归还图书、管理个人资料、查看借阅历史 |

### 2.2 核心功能
#### 2.2.1 图书管理
- **新增图书**：管理员可录入 ISBN、书名、作者、出版社、出版年份、分类、库存数量。
- **编辑/删除图书**：支持修改图书信息，删除时校验是否有未归还的借阅记录。
- **查询图书**：按书名、作者、ISBN、分类等条件搜索，支持分页。
- **批量导入**：通过 CSV/Excel 文件批量添加图书。

#### 2.2.2 读者管理
- **注册/登录**：读者通过邮箱或手机号注册，登录后管理个人信息。
- **个人资料**：查看/修改姓名、联系方式、密码。
- **借阅历史**：查看当前借阅和过去借阅记录。

#### 2.2.3 借阅管理
- **借书**：读者选择图书发起借阅请求，管理员确认或系统自动处理（若库存 >0），生成借阅记录，设置应还日期（如30天）。
- **还书**：读者或管理员操作还书，更新库存，记录实际归还日期。
- **预约**：当图书库存为0时，读者可预约，到书后通知。
- **逾期处理**：系统每日检查应还日期，自动发送逾期提醒（邮件/站内信），并记录逾期天数。

#### 2.2.4 统计报表
- **热门图书**：按借阅次数排序。
- **借阅趋势**：按日/月/年统计借阅数量。
- **读者活跃度**：按借阅次数统计读者。
- **逾期统计**：逾期图书数量、逾期读者排行。

#### 2.2.5 管理员后台
- **用户管理**：查看所有读者，禁用/启用账户。
- **借阅记录管理**：查看所有借阅记录，手动处理异常。
- **系统配置**：设置借阅期限、逾期罚款（可选）、通知模板。

### 2.3 业务流程
1. **借书流程**：读者登录 → 搜索图书 → 查看库存 → 点击借阅 → 系统校验库存和读者违规状态 → 生成借阅记录 → 库存减1 → 返回成功。
2. **还书流程**：读者/管理员进入借阅记录 → 点击还书 → 系统更新记录（归还日期） → 库存加1 → 计算逾期天数（如有）。
3. **预约流程**：读者搜索图书 → 库存为0 → 点击预约 → 系统记录预约 → 当有人还书时，通知预约者 → 预约者一定时间内到馆借阅。

## 3. 非功能性需求
### 3.1 性能
- 并发支持：至少支持 100 个并发用户。
- 响应时间：查询页面 < 2秒，借阅操作 < 1秒。
- 数据量：支持 10 万册图书、1 万读者、50 万条借阅记录。

### 3.2 安全
- 密码加密存储（如 bcrypt）。
- 用户认证：JWT Token，有效期 24 小时。
- 权限控制：管理员和读者角色分离，API 鉴权。
- 防止 SQL 注入、XSS 攻击。

### 3.3 可用性
- 界面简洁，符合常见设计规范（如 Material Design）。
- 支持主流浏览器（Chrome、Firefox、Edge、Safari）。
- 关键操作（借阅、还书）有确认提示。
- 系统可用性 99.5%（每月停机不超过 3.6 小时）。

## 4. 数据模型设计
### 4.1 主要实体及字段
#### 4.1.1 图书（Book）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT (PK) | 自增主键 |
| isbn | VARCHAR(20) | 唯一，ISBN 号 |
| title | VARCHAR(200) | 书名 |
| author | VARCHAR(100) | 作者 |
| publisher | VARCHAR(100) | 出版社 |
| publish_year | INT | 出版年份 |
| category | VARCHAR(50) | 分类（如文学、科技） |
| total_stock | INT | 总库存 |
| available_stock | INT | 可借库存 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

#### 4.1.2 读者（Reader）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT (PK) | 自增主键 |
| email | VARCHAR(100) | 唯一，登录账号 |
| password_hash | VARCHAR(255) | 加密密码 |
| name | VARCHAR(50) | 姓名 |
| phone | VARCHAR(20) | 手机号 |
| status | ENUM('active','disabled') | 账户状态 |
| created_at | DATETIME | 注册时间 |

#### 4.1.3 借阅记录（Borrowing）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT (PK) | 自增主键 |
| reader_id | INT (FK) | 读者ID |
| book_id | INT (FK) | 图书ID |
| borrow_date | DATETIME | 借书日期 |
| due_date | DATETIME | 应还日期 |
| return_date | DATETIME | 实际归还日期（NULL 表示未还） |
| status | ENUM('borrowed','returned','overdue') | 状态 |
| created_at | DATETIME | 记录创建时间 |

#### 4.1.4 预约记录（Reservation）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT (PK) | 自增主键 |
| reader_id | INT (FK) | 读者ID |
| book_id | INT (FK) | 图书ID |
| reserve_date | DATETIME | 预约时间 |
| status | ENUM('pending','notified','fulfilled','cancelled') | 状态 |
| expire_date | DATETIME | 通知后有效期 |

## 5. API 接口设计
### 5.1 认证模块
- `POST /api/auth/register` - 读者注册
- `POST /api/auth/login` - 登录，返回 JWT
- `GET /api/auth/profile` - 获取个人信息（需认证）
- `PUT /api/auth/profile` - 更新个人信息

### 5.2 图书管理
- `GET /api/books` - 分页查询图书（支持搜索参数）
- `GET /api/books/:id` - 获取图书详情
- `POST /api/books` - 新增图书（管理员）
- `PUT /api/books/:id` - 修改图书（管理员）
- `DELETE /api/books/:id` - 删除图书（管理员）
- `POST /api/books/import` - 批量导入（管理员）

### 5.3 借阅管理
- `POST /api/borrowings` - 借书（读者）
- `PUT /api/borrowings/:id/return` - 还书（读者/管理员）
- `GET /api/borrowings/my` - 查看当前读者借阅记录
- `GET /api/borrowings` - 管理员查看所有借阅记录
- `POST /api/reservations` - 预约图书
- `GET /api/reservations/my` - 查看我的预约

### 5.4 统计报表
- `GET /api/reports/popular-books` - 热门图书
- `GET /api/reports/borrowing-trend` - 借阅趋势（支持参数）
- `GET /api/reports/reader-activity` - 读者活跃度
- `GET /api/reports/overdue` - 逾期统计

### 5.5 管理员专属
- `GET /api/admin/readers` - 查看所有读者
- `PUT /api/admin/readers/:id/status` - 禁用/启用读者
- `GET /api/admin/config` - 获取系统配置
- `PUT /api/admin/config` - 更新系统配置

## 6. 技术约束与假设
- **架构**：前后端分离，前端使用 React/Vue，后端使用 Node.js/Spring Boot，数据库使用 MySQL/PostgreSQL。
- **部署**：支持 Docker 容器化部署。
- **假设**：读者和管理员均通过 Web 浏览器访问；系统不处理支付；图书数量有限，不涉及分馆。
- **接口规范**：RESTful API，使用 JSON 格式，JWT 鉴权。

## 7. 验收标准
1. 管理员可成功录入、编辑、删除图书，并支持批量导入。
2. 读者可注册、登录、搜索图书，并发起借阅和还书。
3. 借阅后库存自动更新，逾期自动提醒。
4. 统计报表数据准确，图表展示（可选）。
5. 管理员后台可管理读者和查看所有借阅记录。
6. 系统响应时间满足性能要求，并发测试通过。
7. 安全测试未发现 SQL 注入、XSS 等漏洞。
