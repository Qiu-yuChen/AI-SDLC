# 员工车辆预约系统 规格说明书（V5）

## 1. 项目背景
用户提出需求：建立一个员工车辆预约系统，用于管理公司内部车辆的使用预约。主要使用场景为员工日常通勤。

## 2. 目标用户
- 员工：需要预约车辆。
- 管理员：管理车辆、预约、用户等。

## 3. 核心功能
- 车辆信息管理（添加、编辑、删除车辆）。
- 预约管理（创建、修改、取消预约）。
- 查看可用车辆。
- 预约日历/时间表展示。
- 审批流程：员工提交预约后需经审批人批准。
- 通知功能（待确认具体方式，如邮件、系统内通知）。
- 审计日志记录（与身份认证系统集成）。

## 4. 业务规则
- 时间单位：预约以小时为单位。
- 审批流程：
  - 员工提交预约后，需由指定的审批人（部门经理）审批。
  - 审批状态包括：待审批、已批准、已拒绝。
  - 审批超时处理：超过24小时未审批，系统自动拒绝。
  - 不支持多级审批。
  - 预约被拒绝后，员工可以重新提交预约，且应能查看拒绝原因（由审批人填写）。
- 预约时间冲突：同一车辆在同一时间段内不可被重复预约。
- 员工可以同时预约多辆车，无同一时间段只能有一个有效预约的限制。
- 预约时间限制：
  - 最小预约时长：1小时。
  - 最大提前预约天数：待补充（默认7天？需确认）。
  - 提前预约最小时间：至少提前1小时。
- 车辆状态：可用、已预约、维修中。状态变更由管理员手动设置。
- 身份认证：系统需与公司AD/LDAP集成（或企业微信等），实现统一登录。
- 审计日志：所有关键操作（创建/修改/取消预约、审批、车辆状态变更等）需记录日志，便于追溯。

## 5. 数据实体
- 用户：ID、姓名、部门、联系方式、角色（员工/管理员）、AD账号。
- 车辆：ID、车牌号、品牌、型号、座位数、状态（可用/已预约/维修中）。
- 预约：ID、用户ID、车辆ID、开始时间、结束时间、状态（待审批/已批准/已拒绝/已取消/已完成）。
- 审批记录：ID、预约ID、审批人ID、审批时间、审批意见、状态（通过/拒绝）。
- 审计日志：ID、操作人ID、操作类型、操作时间、操作详情。

## 6. 非功能需求（待补充）

- 系统为Web端，不支持移动端。
- 需与公司AD/LDAP集成，确保身份认证安全。

## 8. 验收标准（待补充）
- 基本预约流程可走通（提交→审批→确认）。
- 冲突检测正确。
- 审批流程正常流转。
- 审批超时自动拒绝功能正常。
- 员工可同时预约多辆车，无冲突。
- 被拒绝后，员工可重新提交并查看原因。
- 管理员可手动设置车辆状态为维修中。
- 集成AD登录正常。
- 审计日志记录完整。
