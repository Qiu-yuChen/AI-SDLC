# 学生管理系统产品规格说明书

## 1. 项目概述

### 1.1 背景
传统学生管理依赖纸质或Excel表格，存在信息分散、更新不及时、查询效率低等问题。为提升学校管理效率，拟开发一套在线学生管理系统。

### 1.2 目标
- 实现学生信息的集中化、数字化管理
- 支持教师/管理员快速查询、新增、修改、删除学生信息
- 提供基础的数据统计功能
- 保证数据安全与权限控制

### 1.3 范围
本项目覆盖学生信息管理、班级管理、用户权限管理、基础数据统计等功能。不涉及课程排课、成绩管理、财务缴费等模块。

## 2. 功能需求

### 2.1 用户角色
| 角色 | 描述 |
|------|------|
| 管理员 | 拥有全部权限，可管理用户、班级、学生信息 |
| 教师 | 可查看、新增、编辑本班学生信息，不可删除 |
| 学生 | 仅可查看个人信息（可选扩展） |

### 2.2 核心功能
1. **学生信息管理**
   - 新增学生（姓名、学号、性别、出生日期、班级、联系方式、家庭住址、照片附件）
   - 编辑学生信息
   - 删除学生（管理员权限）
   - 查询学生（按学号、姓名、班级等字段模糊搜索）
   - 查看学生详情（包括照片）
   - 支持上传学生照片附件（格式：jpg/png，大小不超过5MB）
2. **班级管理**
   - 新增/编辑/删除班级
   - 班级列表展示
3. **用户管理**
   - 管理员创建教师账号（仅支持统一创建，不支持学生自主注册）
   - 分配角色与权限
4. **数据统计**
   - 按班级统计学生人数
   - 性别比例饼图
   - 导出学生列表（Excel）

### 2.3 业务流程
1. **管理员登录** → 进入系统 → 管理班级/用户/学生
2. **教师登录** → 查看本班学生列表 → 新增/编辑学生 → 查询
3. **学生登录（可选）** → 查看个人信息

## 3. 非功能性需求

### 3.1 性能
- 页面响应时间 < 2秒（正常网络）
- 支持并发用户数 ≥ 200
- 学生数据量 ≤ 10万条时，查询延迟 < 1秒

### 3.2 安全
- 用户密码加密存储（bcrypt）
- 基于JWT的登录态管理
- 接口权限校验（RBAC）
- 防止SQL注入、XSS攻击
- 所有操作记录日志

### 3.3 可用性
- 界面简洁，符合Web标准
- 支持主流浏览器（Chrome、Firefox、Edge）
- 响应式设计，适配平板与PC
- 提供操作反馈与错误提示

## 4. 数据模型设计

### 4.1 学生表 (students)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT (PK) | 自增主键 |
| student_no | VARCHAR(20) | 学号，唯一索引 |
| name | VARCHAR(50) | 姓名 |
| gender | ENUM('男','女') | 性别 |
| birth_date | DATE | 出生日期 |
| class_id | INT (FK) | 班级ID |
| phone | VARCHAR(20) | 联系方式 |
| address | VARCHAR(200) | 家庭住址 |
| photo_url | VARCHAR(255) | 照片存储路径（可选） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 4.2 班级表 (classes)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT (PK) | 自增主键 |
| name | VARCHAR(50) | 班级名称，如“2024级1班” |
| grade | VARCHAR(20) | 年级 |
| teacher_id | INT (FK) | 班主任教师ID |

### 4.3 用户表 (users)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT (PK) | 自增主键 |
| username | VARCHAR(50) | 用户名，唯一 |
| password | VARCHAR(255) | 加密后的密码 |
| role | ENUM('admin','teacher','student') | 角色 |
| teacher_id | INT (FK) | 关联教师信息（可选） |

## 5. API 接口设计

### 5.1 认证模块
- `POST /api/login` - 用户登录，返回JWT token
- `POST /api/logout` - 登出

### 5.2 学生管理
- `GET /api/students` - 查询学生列表（支持分页、搜索）
- `GET /api/students/{id}` - 获取学生详情
- `POST /api/students` - 新增学生（支持文件上传）
- `PUT /api/students/{id}` - 编辑学生（支持文件上传）
- `DELETE /api/students/{id}` - 删除学生（仅管理员）
- `POST /api/students/upload-photo` - 上传学生照片附件

### 5.3 班级管理
- `GET /api/classes` - 班级列表
- `POST /api/classes` - 新增班级
- `PUT /api/classes/{id}` - 编辑班级
- `DELETE /api/classes/{id}` - 删除班级

### 5.4 统计
- `GET /api/statistics/class-count` - 各班人数统计
- `GET /api/statistics/gender-ratio` - 性别比例

### 5.5 导出
- `GET /api/export/students` - 导出学生Excel

## 6. 技术约束与假设

### 6.1 技术栈
- 前端：Vue 3 + Element Plus
- 后端：Spring Boot 2.x + MyBatis-Plus
- 数据库：MySQL 8.0
- 认证：JWT
- 部署：Docker容器化
- 文件存储：本地文件系统或对象存储（如MinIO）

### 6.2 假设
- 用户已具备基本的计算机操作能力
- 学校网络环境稳定
- 初始数据由管理员手动导入
- 不与现有教务系统或学校数据库进行数据同步
- 学生账号由管理员或教师统一创建，不支持学生自主注册

## 7. 验收标准

1. 所有核心功能（增删改查、权限控制）可正常使用
2. 搜索功能支持模糊匹配，响应时间 < 1秒
3. 管理员可以创建/删除教师账号
4. 教师只能操作本班学生，无法看到其他班级
5. 导出Excel文件格式正确，包含所有学生字段
6. 密码加密存储，JWT有效期为24小时
7. 系统在Chrome、Firefox、Edge上表现一致
8. 接口返回标准JSON格式，错误码清晰
9. 学生照片附件上传功能正常，支持jpg/png格式，大小限制5MB
10. 学生详情页可显示照片
