# 员工临时车辆预约系统产品规格说明书

---

## 第一章：系统概述

### 1.1 项目背景  
随着公司业务发展，员工日常通勤需求增加，为提升员工出行效率、优化车辆资源利用率，特开发“员工临时车辆预约系统”。该系统旨在为员工提供便捷、安全、透明的临时用车服务，实现车辆资源的合理分配与使用管理。

### 1.2 系统目标  
- 实现员工临时用车的在线预约、审批、使用、归还全流程管理  
- 提高车辆使用效率，减少车辆闲置  
- 增强员工用车体验，降低管理成本  
- 支持多部门、多岗位员工的灵活用车需求  

### 1.3 系统范围  
- 覆盖公司内部员工（含临时工、实习生）  
- 支持车辆类型：私家车、公务车、共享车（可选）  
- 预约周期：1小时至72小时（可配置）  
- 支持移动端（App/微信小程序）与PC端双端访问  

### 1.4 系统用户角色  
| 角色 | 权限说明 |
|------|----------|
| 员工 | 提交用车申请、查看预约状态、确认用车、归还车辆 |
| 部门主管 | 审批员工用车申请 |
| 行政/车辆管理员 | 管理车辆信息、审批权限、查看用车记录、处理异常 |
| 系统管理员 | 系统配置、用户管理、日志审计、数据导出 |

## 第二章：功能需求

### 2.1 员工端功能

#### 2.1.1 用车预约  
- 员工可选择用车时间（起止时间，支持小时级、分钟级）  
- 可选择用车类型（私家车、公务车、共享车）  
- 填写用车目的（如：出差、加班、接送孩子）  
- 上传行程说明（可选）  
- 提交申请后进入待审批状态  

#### 2.1.2 用车审批流程  
- 支持单级审批（部门主管）或多级审批（如：部门主管 → 行政部）  
- 支持审批意见填写（通过/驳回/修改）  
- 审批结果实时通知员工  

#### 2.1.3 预约状态查看  
- 实时查看预约状态（待审批、已通过、已拒绝、已用车、已归还）  
- 查看历史用车记录（按时间、类型、状态筛选）  

#### 2.1.4 用车确认与归还  
- 用车开始前，员工需确认车辆信息（车牌、位置、司机）  
- 归还车辆时，系统自动记录归还时间、地点、状态  
- 归还后自动更新用车记录  

#### 2.1.5 通知提醒  
- 提前1小时提醒员工审批状态  
- 提前15分钟提醒用车开始  
- 归还车辆后发送完成通知  

---

### 2.2 管理端功能

#### 2.2.1 车辆管理  
- 添加/编辑/删除车辆信息（车牌、车型、所属部门、状态、位置）  
- 设置车辆可用时间范围（如：每日8:00-18:00）  
- 车辆状态管理（空闲、使用中、维修中、不可用）  

#### 2.2.2 审批流程配置  
- 配置审批规则（如：用车时长>4小时需主管+行政双签）  
- 支持自定义审批节点、审批人、审批顺序  

#### 2.2.3 用车记录查询  
- 支持按时间、员工、车辆、状态、部门等条件筛选  
- 支持导出Excel/CSV格式  
- 提供用车统计图表（如：月度用车趋势、部门用车占比）  

#### 2.2.4 异常处理  
- 支持手动标记“异常用车”（如：未归还、超时、违规）  
- 异常记录可追溯至具体员工与审批人  

#### 2.2.5 数据报表  
- 月度用车统计（总次数、平均时长、用车高峰时段）  
- 部门用车对比  
- 车辆使用率分析

## 第三章：非功能需求

### 3.1 性能要求  
- 系统响应时间：页面加载 ≤ 2秒，操作响应 ≤ 1秒  
- 支持并发用户数 ≥ 500  
- 预约提交成功率 ≥ 99.9%  

### 3.2 安全性要求  
- 用户登录采用 HTTPS + 双因素认证（可选）  
- 敏感操作（如审批、车辆修改）需二次确认  
- 数据加密存储（用户密码、车牌信息）  
- 审计日志记录所有关键操作（登录、审批、修改）  

### 3.3 可用性要求  
- 支持主流浏览器（Chrome、Edge、Firefox）  
- 移动端适配（H5/小程序）  
- 支持离线缓存（预约信息）  

### 3.4 可维护性  
- 模块化设计，便于功能扩展  
- 提供API接口供第三方系统集成（如OA、ERP）  
- 日志系统支持错误追踪与报警  

### 3.5 兼容性  
- 支持 Windows、macOS、iOS、Android  
- 支持主流浏览器（Chrome、Edge、Safari）

## 第四章：数据模型

### 4.1 核心实体关系图（ERD）

```text
员工 (Employee)
├── 员工ID (PK)
├── 姓名
├── 部门
├── 职位
├── 手机号
└── 邮箱

车辆 (Vehicle)
├── 车辆ID (PK)
├── 车牌号
├── 车型
├── 所属部门
├── 状态（空闲/使用中/维修中）
└── 位置（可选）

用车申请 (Reservation)
├── 申请ID (PK)
├── 员工ID (FK)
├── 车辆ID (FK)
├── 用车时间（开始、结束）
├── 用车类型
├── 用车目的
├── 行程说明
├── 提交时间
├── 审批状态（待审批/已通过/已拒绝）
├── 审批人
├── 审批时间
└── 备注

用车记录 (UsageLog)
├── 记录ID (PK)
├── 申请ID (FK)
├── 开始时间
├── 结束时间
├── 实际使用时长
├── 归还状态（已归还/未归还）
└── 归还时间
```

### 4.2 数据存储  
- 数据库：MySQL 8.0（主库） + Redis（缓存）  
- 存储结构：按部门、时间分区，支持快速查询

## 第五章：系统架构设计

### 5.1 架构图（简要）

```text
用户端（Web/小程序）
    ↓
API网关（Nginx）
    ↓
微服务架构：
├── 用户服务（用户管理、登录）
├── 车辆服务（车辆信息、状态）
├── 预约服务（申请、审批、状态）
├── 审批服务（流程引擎）
├── 通知服务（短信/站内信）
└── 数据服务（报表、统计）

数据库层：
├── MySQL（主数据）
└── Redis（缓存、会话）

外部系统集成：
→ OA系统（审批流程）
→ 企业微信（通知推送）
→ GPS定位（车辆位置追踪，可选）
```

### 5.2 技术栈  
| 模块 | 技术选型 |
|------|----------|
| 前端 | Vue3 + Element Plus（PC）、Taro（小程序） |
| 后端 | Spring Boot（Java）或 Node.js |
| 数据库 | MySQL 8.0 |
| 缓存 | Redis |
| 消息队列 | RabbitMQ（异步通知） |
| 部署 | Docker + Kubernetes（可选） |
| 安全 | JWT + OAuth2.0 |
