# 员工临时车辆预约系统产品规格说明书

---

## 第一章：系统概述

### 1.1 项目背景  
随着公司业务发展，员工日常出行需求增加，为提升员工通勤效率、优化车辆使用管理，特开发“员工临时车辆预约系统”。该系统旨在实现员工对临时车辆的在线预约、审批、使用与归还的全流程管理，提高车辆使用透明度与资源利用率。

### 1.2 系统目标  
- 实现员工在线提交临时车辆预约申请  
- 支持部门负责人或管理员进行审批  
- 提供车辆状态实时查询与使用记录  
- 支持预约记录的统计分析与报表生成  
- 提高车辆使用效率，减少资源浪费  

### 1.3 系统范围  
- 覆盖公司内部员工（含临时工、实习生）  
- 支持公司内所有可用车辆（如行政车、公务车、共享车）  
- 预约周期为1小时至24小时，最长不超过72小时  
- 适用于公司办公区、园区、指定停车场  

### 1.4 系统用户角色  
| 角色 | 权限说明 |
|------|----------|
| 员工 | 提交预约申请、查看审批状态、查看已预约车辆信息 |
| 部门负责人 | 审批员工预约申请、查看本部门预约情况 |
| 行政管理员 | 管理车辆信息、审批权限设置、查看全局数据、生成报表 |
| 系统管理员 | 系统配置、用户管理、日志审计、数据备份 |

## 第二章：功能需求

### 2.1 员工预约功能  
- 员工登录后可查看可用车辆列表  
- 选择车辆类型（如行政车、公务车、共享车）  
- 填写预约时间（起止时间，支持1小时至72小时）  
- 填写使用目的（如出差、会议、接送）  
- 提交申请后进入待审批状态  
- 支持预约取消（在审批前）  
- 支持预约修改（在审批前）  

### 2.2 审批流程  
- 员工提交后，系统自动发送审批通知至部门负责人  
- 部门负责人可在审批界面进行“通过”或“拒绝”操作  
- 拒绝后系统通知员工并记录原因  
- 审批通过后，车辆状态更新为“已预约”  
- 支持审批流程设置（如多级审批、自动转交）  

### 2.3 车辆管理  
- 管理员可维护车辆信息（车牌号、车型、状态、位置、所属部门）  
- 车辆状态包括：空闲、已预约、正在使用、已归还  
- 支持车辆位置更新（通过GPS或手动录入）  
- 支持车辆维护提醒（如定期保养）  

### 2.4 使用与归还  
- 预约成功后，员工可获取车辆使用码（二维码）  
- 使用过程中需在系统中记录实际使用时间  
- 使用结束前需在系统中提交归还申请  
- 系统自动校验归还时间是否在预约范围内  
- 归还后车辆状态更新为“空闲”  

### 2.5 查询与统计  
- 员工可查看个人预约历史  
- 部门负责人可查看本部门预约情况  
- 管理员可查看全局车辆使用率、热门时段、高频使用部门等  
- 支持导出Excel报表（预约记录、使用统计、异常情况）  

### 2.6 系统通知  
- 提交申请后发送短信/邮件通知负责人  
- 审批结果通知员工  
- 预约即将到期（提前1小时）发送提醒  
- 归还提醒（提前15分钟）

## 第三章：非功能需求

### 3.1 性能要求  
- 系统响应时间：页面加载 ≤ 2秒，操作响应 ≤ 1秒  
- 支持并发用户数 ≥ 500  
- 预约提交成功率 ≥ 99.9%  

### 3.2 安全性要求  
- 用户登录采用双因素认证（短信+密码）  
- 敏感操作（如审批、删除）需二次确认  
- 数据传输采用 HTTPS 加密  
- 用户信息加密存储（AES-256）  
- 日志记录所有关键操作（登录、审批、修改）  

### 3.3 可用性要求  
- 系统全年可用率 ≥ 99.9%  
- 支持移动端（H5或小程序）访问  
- 支持多语言（中文为主，可扩展英文）  

### 3.4 兼容性要求  
- 浏览器兼容：Chrome、Edge、Firefox（最新版本）  
- 移动端兼容：iOS 12+，Android 8.0+  
- 支持离线缓存（预约提交缓存）  

### 3.5 可维护性  
- 采用模块化设计，便于功能扩展  
- 提供完整的API文档与日志接口  
- 支持热部署与灰度发布

## 第四章：数据模型

### 4.1 核心实体关系图（ERD）

```plaintext
员工 (Employee)
├── 员工ID (PK)
├── 姓名
├── 部门
├── 职位
├── 联系方式
└── 登录账号

车辆 (Vehicle)
├── 车辆ID (PK)
├── 车牌号
├── 车型
├── 所属部门
├── 当前状态（空闲/已预约/使用中/已归还）
├── 位置（GPS坐标或停车场编号）
└── 创建时间

预约记录 (Reservation)
├── 预约ID (PK)
├── 员工ID (FK)
├── 车辆ID (FK)
├── 起始时间
├── 结束时间
├── 使用目的
├── 提交时间
├── 审批状态（待审批/已通过/已拒绝）
├── 审批人
├── 审批时间
└── 备注

归还记录 (Return)
├── 归还ID (PK)
├── 预约ID (FK)
├── 归还时间
├── 归还人
└── 备注

审批记录 (Approval)
├── 审批ID (PK)
├── 预约ID (FK)
├── 审批人
├── 审批时间
├── 审批结果
└── 审批意见
```

### 4.2 数据表字段说明（部分）

| 表名 | 字段 | 类型 | 说明 |
|------|------|------|------|
| employee | employee_id | BIGINT | 主键 |
| employee | name | VARCHAR(50) | 姓名 |
| vehicle | license_plate | VARCHAR(20) | 车牌号 |
| reservation | start_time | DATETIME | 预约开始时间 |
| reservation | end_time | DATETIME | 预约结束时间 |
| reservation | status | ENUM('pending','approved','rejected') | 状态 |

## 第五章：系统架构设计

### 5.1 技术架构  
- 前端：Vue.js + Element UI（PC端） + H5（移动端）  
- 后端：Spring Boot（Java） + MyBatis  
- 数据库：MySQL 8.0（主从复制）  
- 消息队列：RabbitMQ（用于通知推送）  
- 文件存储：阿里云OSS（用于上传附件）  
- 部署方式：Docker + Nginx + Kubernetes（可选）  

### 5.2 系统部署  
- 部署环境：公司内网（私有云）  
- 访问方式：内网IP或公司门户入口  
- 网络安全：防火墙策略、IP白名单、访问控制  

### 5.3 接口设计（示例）  
```json
POST /api/reservation/submit
{
  "employeeId": "E001",
  "vehicleId": "V001",
  "startTime": "2025-04-05T09:00:00",
  "endTime": "2025-04-05T10:00:00",
  "purpose": "会议",
  "remark": "需要接送客户"
}
```

## 第六章：用户界面原型（概要）

### 6.1 员工端界面  
- 登录页 → 首页 → 预约申请页 → 我的预约页  
- 首页展示：可用车辆列表、今日热门车辆、预约提醒  
- 预约申请页：选择车辆、填写时间、目的、提交  

### 6.2 部门负责人端  
- 审批列表页：按部门、时间筛选  
- 审批详情页：查看申请内容、审批操作按钮  

### 6.3 管理员端  
- 车辆管理页：增删改查  
- 统计报表页：使用率、时段分布、部门对比
