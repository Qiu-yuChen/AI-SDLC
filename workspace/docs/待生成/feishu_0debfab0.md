# 员工临时车辆预约系统产品规格说明书

---

## 第一章：系统概述

### 1.1 项目背景  
随着公司业务发展，员工日常通勤需求增加，为提升员工出行效率、优化车辆资源利用率，特开发“员工临时车辆预约系统”。该系统旨在为员工提供便捷、透明、高效的临时车辆使用服务，实现车辆资源的合理调配与使用监管。

### 1.2 系统目标  
- 实现员工临时用车的在线预约与管理  
- 提高车辆使用效率，减少空置率  
- 增强员工出行便利性与满意度  
- 支持灵活的用车时间、地点、车型选择  
- 提供完整的用车记录与审批流程  

### 1.3 系统范围  
本系统覆盖以下核心功能模块：  
- 员工端：预约申请、用车记录查看、通知提醒  
- 管理端：车辆管理、审批流程、用车统计、异常监控  
- 系统支持移动端（App/微信小程序）与PC端双平台  

### 1.4 目标用户  
- 员工（普通用户）  
- 行政/人事部门（审批人员）  
- 车辆管理专员（车辆维护与调度）

## 第二章：系统功能需求

### 2.1 员工端功能

#### 2.1.1 用车预约  
- 支持选择用车时间（起止时间，支持1小时以上）  
- 支持选择用车地点（支持输入或从地图选择）  
- 支持选择车型（如：小型车、SUV、商务车）  
- 支持填写用车事由（如：出差、会议、接送）  
- 支持上传附件（如：出差证明、行程单）  
- 预约提交后自动进入待审批状态  

#### 2.1.2 用车记录查询  
- 按时间、状态、车型筛选历史用车记录  
- 查看用车详情（时间、地点、车型、事由、审批状态）  
- 支持导出用车记录（Excel格式）  

#### 2.1.3 通知提醒  
- 提前1小时提醒用户预约即将开始  
- 提前2小时提醒审批人员待审批事项  
- 提供短信/站内信/App推送通知  

#### 2.1.4 个人中心  
- 查看个人用车记录  
- 修改密码、绑定手机号  
- 个人资料维护（姓名、部门、工号）  

### 2.2 管理端功能

#### 2.2.1 车辆管理  
- 添加/编辑/删除车辆信息（车牌号、车型、颜色、状态、所属部门）  
- 设置车辆可用时间段（如：每日8:00-18:00）  
- 设置车辆可用车位（支持多地点）  
- 车辆状态监控（空闲/占用/维修中）  

#### 2.2.2 审批流程管理  
- 配置审批规则（如：单次用车≤1小时，需部门主管审批；>1小时需HR+部门主管双签）  
- 支持多级审批（可自定义审批路径）  
- 审批意见填写与历史记录查看  
- 审批状态实时更新（待审批、已通过、已拒绝）  

#### 2.2.3 统计报表  
- 按部门、时间、车型统计用车频次  
- 用车高峰时段分析  
- 车辆使用率统计（空闲率、使用率）  
- 异常用车预警（如：超时、重复预约）  

#### 2.2.4 异常处理  
- 支持手动标记“用车异常”（如：未按时归还、临时取消）  
- 异常记录可追溯，支持导出  
- 提供用车超时提醒（超时30分钟自动提醒）  

#### 2.2.5 系统设置  
- 配置通知模板（短信、站内信）  
- 设置节假日用车规则（如：节假日不可用车）  
- 配置系统参数（如：最大预约时长、最小预约时长）

## 第三章：非功能需求

### 3.1 性能要求  
- 系统响应时间：页面加载 ≤ 2秒，操作响应 ≤ 1秒  
- 支持并发用户数 ≥ 500人  
- 支持日均预约请求 ≥ 1000次  

### 3.2 安全性要求  
- 用户登录采用 HTTPS + 双因素认证（可选）  
- 敏感操作（如删除车辆、审批）需二次确认  
- 数据加密存储（用户密码、敏感信息）  
- 审计日志记录所有关键操作（登录、审批、修改）  

### 3.3 可用性要求  
- 系统全年可用率 ≥ 99.9%  
- 支持7×24小时服务（含节假日）  
- 提供错误提示与操作引导  

### 3.4 兼容性要求  
- 支持主流浏览器：Chrome、Edge、Firefox、Safari  
- 支持移动端：iOS 12+，Android 8.0+  
- 支持微信小程序（兼容微信版本 ≥ 7.0）  

### 3.5 可维护性  
- 模块化设计，便于后期功能扩展  
- 提供API接口，支持与OA、HR系统集成  
- 日志系统支持问题快速定位

## 第四章：数据模型设计

### 4.1 核心数据表

| 表名 | 字段说明 |
|------|----------|
| `users` | id, name, department, phone, role, status, create_time |
| `vehicles` | id, license_plate, model, color, status, location, create_time |
| `appointments` | id, user_id, start_time, end_time, location, reason, status, create_time, approve_status, approve_user_id, approve_time |
| `approval_rules` | id, rule_type, condition, approval_level, approval_path |
| `logs` | id, action, user_id, target, detail, timestamp |

### 4.2 数据关系  
- 员工 → 预约（1:N）  
- 车辆 → 预约（N:1）  
- 预约 → 审批（1:1）  
- 审批 → 审批人（1:N）

## 第五章：系统架构设计

### 5.1 架构图（文字描述）  
```
用户端（App/PC/小程序）
     ↓
HTTP API Gateway（Nginx）
     ↓
微服务架构：
  - 用户服务（User Service）
  - 预约服务（Appointment Service）
  - 审批服务（Approval Service）
  - 车辆服务（Vehicle Service）
  - 报表服务（Report Service）
     ↓
数据库（MySQL主从集群）
     ↓
消息队列（RabbitMQ）用于异步通知
     ↓
Redis缓存（用户会话、热门车辆信息）
```

### 5.2 技术栈  
- 前端：Vue3 + Element Plus（PC端），Taro（小程序）  
- 后端：Spring Boot（Java）或 Node.js（可选）  
- 数据库：MySQL 8.0（主从复制）  
- 消息中间件：RabbitMQ  
- 缓存：Redis  
- 部署：Docker + Kubernetes（可选）

## 第六章：用户界面原型（概要）

### 6.1 员工端界面  
- **首页**：显示今日可用车辆、待审批事项、最近用车记录  
- **预约页**：时间选择器、地点地图、车型选择、事由填写、提交按钮  
- **我的用车**：列表展示历史用车，支持筛选与导出  

### 6.2 管理端界面  
- **车辆管理页**：车辆列表，支持新增、编辑、状态切换  
- **审批中心**：待审批列表，支持点击查看详情、审批通过/拒绝  
- **统计报表页**：图表展示使用趋势、部门分布、车辆利用率
