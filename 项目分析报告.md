## 一、项目概述

AI-SDLC 是一套基于 **CrewAI 多 Agent 流水线** 的 IT 功能全链路自动化开发系统。通过 Web UI 上传产品规格说明书，系统自动完成 **概要设计 → 代码生成 → 单元测试** 三阶段开发。

| 指标 | 数值 |
|------|------|
| 后端框架 | FastAPI + Uvicorn |
| Agent 框架 | CrewAI 1.14.5 |
| LLM 网关 | LiteLLM 1.86.2 |
| 前端框架 | React 18 + TypeScript + Vite + Tailwind |
| 后端源码行数 | ~1,600 行 Python（本次新增 ~480 行） |
| 前端源码行数 | ~1,200 行 TSX/CSS（完全重写） |

## 四、新增功能详解

### 4.1 提示词优化（核心创新）

解决用户"不知道怎么写产品规格说明书"的痛点：

```
用户输入："做一个员工预约系统"
      ↓ POST /api/prompt/optimize
AI 生成完整 Markdown 规格书（6-7 个章节）
      ↓
AI 追问 2-3 个问题（气泡式对话）
      ↓ 用户回答
POST /api/prompt/refine → AI 更新规格书 → 新一轮追问
      ↓
用户点击「直接导入并开始流程」
      ↓
自动创建批次 → 启动流水线 → 跳转监控页
```

### 4.2 质量检查机制

- **审查 Agent**：执行 LLM 审查各节点产出物，返回 `{score, total, issues, suggestions}`
- **工具链**：`lint_check`(flake8) → `coverage_check`(pytest-cov) → `calculate_quality_score`
- **熔断层**：LLM 审查失败时 fallback 到工具打分
- **前端展示**：节点卡片底部绿色渐变质量进度条 + 数字百分比

## 六、优点

1. **端到端自动化已跑通**：从上传 Markdown → 设计 → 代码 → 测试，150 个用例全部通过，证明了 Agent 协作的有效性

3. **React 实时反馈**：WebSocket 驱动的 PipelineView + ReActLog 让用户实时看到 Agent 思考过程
4. **深色主题**：暗色界面更专业，符合开发者审美
5. **节点级质量评分**：自动化 + 手动熔断的评分机制，帮助用户判断 Agent 产出质量
6. **手动/自动双模式**：支持全自动流转和逐步执行，灵活度好
